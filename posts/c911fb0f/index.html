碎碎念

虽然vibe coding已经火了一段时间（并非），优秀的教程也有很多，但是在这里还是想要谈一下自己的一些看法：如何高效地进行vibe coding

作者才疏学浅，如果有问题还请多多指正

精准提示工程

  • 先描述架构再写代码：先用自然语言说明整体架构、模块划分和数据流，再让AI生成具体的实现
  • 分步迭代：从简单功能开始，逐步增加复杂度，避免一次性要求完整应用
  • 提供上下文：明确技术栈、框架版本、已有代码结构，减少AI猜测

    省流：一定要先写文档再开发

建立适合人机协作的工作流

需求描述 → AI生成 → 人工审查 → 反馈修正 → 测试验证 → 迭代优化

  • 保持主导权：用伪代码（pseudo-code）先表达逻辑，再让LLM翻译成实际代码，确保你始终掌控设计思路
  • 验收而非盲信：AI生成的代码必须经过人工审查、测试和调试，不能直接部署

    省流：你来带头走，AI只是辅助

工具链选择

  • IDE集成：GitHub Copilot、Cursor、Replit Ghostwriter 等提供无缝的对话式编码体验
  • 专用平台：Google AI Studio（Vibe Code with Gemini）、Supabase等提供端到端的AI辅助开发环境
  • 多模型组合：根据任务类型切换不同AI模型（如Claude适合长上下文，GPT适合创意，CodeLlama适合特定语言）

    省流：氪点金还是有用

避免常见的陷阱

  • ❌ 不要期望”一次成功”：AI生成的代码常有逻辑错误、安全漏洞或性能问题
  • ❌ 避免模糊指令：如”做个好看的网站”应改为”用React+Tailwind创建一个包含导航栏、产品卡片和联系表单的响应式单页应用”
  • ✅ 保存迭代过程：记录每次prompt和生成结果，便于回溯和复用

    省流：别指望一遍过，多改改

认清现实

  • 适合场景：原型开发、样板代码、CRUD操作、简单算法实现
  • 需谨慎场景：核心业务逻辑、安全敏感代码、性能关键路径、复杂算法设计
  • 必备基础：即使使用vibe coding，仍需具备基础编程知识以审查和调试AI生成的代码

    省流：玩玩就行，不要太当回事

💡 关键心态转变：Vibe Coding 不是”取代程序员”，而是将开发者从重复性编码中解放，聚焦于问题定义、架构设计和质量把控——从”写代码的人”升级为”指挥AI创造价值的人”。
